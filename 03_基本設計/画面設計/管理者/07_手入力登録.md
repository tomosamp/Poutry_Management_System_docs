# 手入力登録画面

## 1. 画面概要
- 目的: CSV提出が困難な生産者に対し、職員が画面上で月次データを直接入力できるようにする。
- 主利用者: 食鳥協会職員。
- 画面カテゴリ: データ提出。

## 2. 主な機能
1. 生産者選択（検索・フィルタ）。
2. 鳥種別に必要な入力項目を切り替え表示。
3. 月次メモ（全体メモ）および項目メモ（任意）の入力。
4. 入力確認画面へ遷移し確定登録。

## 3. 画面要素
| ID | セクション | 主な項目 |
| --- | --- | --- |
| SEC-MAN-01 | 生産者指定カード | 生産者検索/選択（サイドバー起点は必須）。生産者一覧/詳細からの遷移時は選択済み状態で表示。 |
| SEC-MAN-02 | 基本情報 | 提出月（YYYY-MM）、担当職員（自動表示）、鳥種（生産者から自動表示）、月次メモ（任意）。 |
| SEC-MAN-03 | 入力フォーム（ブロイラー） | 先月実績: ひな入雛羽数（羽）/生体処理羽数（羽）/生体処理重量（kg）/生体処分別実羽数（禁止・全部廃棄・一部廃棄：羽）。提出月〜+2ヶ月予定: ひな入雛羽数（羽）/生体処理羽数（羽）/生体処理重量（kg）。各入力項目に対して「項目メモ（任意）」を併設。 |
| SEC-MAN-04 | 入力フォーム（地鶏/成鶏） | 鳥種に応じた必要項目（先月実績のみ）。各入力項目に対して「項目メモ（任意）」を併設。 |
| SEC-MAN-05 | バリデーション結果 | 警告・エラーのリスト表示。 |
| BTN-MAN-01 | 確認へ進む | ボタン | 
| BTN-MAN-02 | クリア | ボタン | 入力値を初期化。 |
| BTN-MAN-03 | キャンセル | ボタン | 生産者一覧へ戻る。 |

## 4. 入力・検証
- 数値項目: ひな入雛羽数/生体処理羽数は羽（整数）で入力・保存する。重量はkg単位で入力し、DBはkg（整数）で保持する。
- 生体処分別実羽数（禁止・全部廃棄・一部廃棄）は羽単位（整数）で入力・保存する。
- 負値禁止（サーバ側バリデーション）。
- 予定値未入力はエラー（ブロイラーのみ）。
- 追加の相関チェック（例: 入雛羽数と処理羽数の関係など）は将来検討。
- 提出月の初期値は「当日が16日以降なら翌月、1日〜15日なら当月」を既定とする。
- 初期表示値（最新値）が入っている前提で、値を変更せず項目メモだけを残す運用を許容する。

## 5. 遷移
- 確認へ進む → 入力確認画面。
- クリア → 入力初期化。
- キャンセル → 生産者一覧。

## 6. 備考
- 月次メモ（全体メモ）と項目メモは提出履歴詳細でも表示され、関係機関向けには出力しない。
